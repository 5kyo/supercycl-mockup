01

행사 구조 및 체험 개요

4/11 행사에서 유스메타 측(최문창 상무)이 Supercycl 협업을 발표하고 시연합니다. 유저들이 현장에서 직접 모바일로 가입부터 거래까지 체험하는 것이 목표입니다. PC 시연과 모바일 체험은 별도 트랙으로 운영됩니다.

PC 시연 (상용 환경) PC

최문창 상무 발표용
  • 상용 Supercycl 웹 사용
  • CEX 다중 연동 및 실제 거래 시연
  • BingX, Binance, OKX API 연동 데모
  • 노트북 지참 유저: 직접 API 연동 체험 가능

모바일 체험 (임시 버전) MOBILE

1,000명+ 유저 체험용
  • 임시 모바일 웹 (이벤트 후 폐기 처분)
  • Hyperliquid Testnet 기반 (실제 자금 X)
  • 가입 시 지갑 자동 생성 + 테스트 자금 자동 지급
  • 상용 PC 버전과 완전히 분리된 환경
  • 가입 시 유스메타 레퍼럴 코드 자동 등록
✓ 핵심 전제: 지갑 자동 생성 현재 Supercycl은 회원가입 시 지갑이 자동으로 생성되는 구조입니다. 따라서 유저가 별도로 MetaMask를 설치하거나 지갑을 Import할 필요가 없습니다. 자동 생성된 지갑에 테스트넷 자금을 미리 넣어두면, 가입 즉시 거래가 가능합니다.
⚠ 주의: 임시 환경 이번 모바일 버전은 이벤트 전용 임시 빌드입니다. 정식 모바일 버전과는 별개로 설계하며, 세미나 종료 후 폐기될 가능성이 높습니다. 단, 모바일로 가입하더라도 유스메타 레퍼럴 코드는 등록됩니다.
02

유스메타 측 요구사항

두찬실장님과의 협의를 통해 확인된 유스메타 측 요구사항입니다.

필수
가입 → 연동 → 거래 풀 플로우

발표 중 유저에게 QR 코드를 공유하고, 현장에서 바로 가입부터 거래까지 체험시키는 것이 핵심. 시연을 보러 모인 유저들이 직접 가입하게 하여 가입률을 높이는 것이 목표.

필수
2배 레버리지 고정 체험

유스메타 파트너 버전의 핵심 기능. 유저 보호 정책에 따라 최대 레버리지가 2배로 제한되는 기능을 체험 플로우에 포함. Adjust Leverage 모달에서 2배 초과 선택 불가 UI 포함.

필수
자동 TP/SL 기능 체험

사전 설정한 TP/SL 전략(%)이 주문 체결 시 자동 적용되는 기능. Settings 또는 Order Form에서 설정 → 주문 시 자동 TP/SL 생성 → 포지션에서 확인하는 흐름 체험.

필수
유스메타 레퍼럴 코드 등록

임시 모바일 버전이라도 가입 시 유스메타 레퍼럴 코드(partner_code = YOUTHMETA)가 자동 등록되어야 함. 가입 자체가 유스메타 측의 주요 성과 지표.

희망
사전 테스트 시간 확보

유스메타 측(최문창 상무 포함)이 행사 전에 프로세스를 실제 테스트해보고 자료를 만들어야 함. 최소 2~3일 전 테스트 환경 제공 필요.

참고
PC 유저 대상 CEX API 연동 시연

노트북 지참 유저에게는 BingX, Binance, OKX 등 실제 API 연동을 시연할 예정. 이 부분은 상용 PC 버전에서 진행하며 모바일 체험과 별도.

💡 유스메타 측 확인 필요 사항 ① 자동 TP/SL 기본 상태: 기본 ON일 경우 Default TP/SL 수치 필요    ② 최초 로그인 시 레버리지 2배 고정 안내 팝업 문구 영문/국문 여부    ③ 추후 CEX 연동 가능하다는 안내 문구 포함 여부

04

테스트넷 자금 처리 방안

지갑이 자동 생성되는 구조이므로, Supercycl 백엔드에서 테스트넷 자금을 자동으로 지급하는 방식이 가장 적합합니다.

⚠ HL Testnet Faucet 제약사항 Hyperliquid 테스트넷 Faucet(1,000 mock USDC 지급)은 해당 지갑이 메인넷에서 최소 $5 USDC를 입금한 이력이 있어야 사용 가능합니다. 자동 생성 지갑은 메인넷 이력이 없으므로 Faucet을 직접 사용할 수 없습니다.

권장 방식: 마스터 지갑 → 유저 지갑 Transfer

단계 내용 시점
1 Supercycl 마스터 지갑 다수를 메인넷에서 활성화 ($5 USDC 입금) 사전 준비 (D-7)
2 각 마스터 지갑에서 Testnet Faucet claim (1,000 USDC × N개) 사전 준비 (D-5~3)
3 유저 가입 시 자동 생성된 지갑으로 mock USDC 자동 전송 실시간 (가입 즉시)
4 유저 화면에 잔고 표시 → 거래 시작 가능 전송 후 ~10초
필요 마스터 지갑 수 추정 Faucet claim은 주소당 1회(1,000 USDC). 유저 1,000명 × 1,000 USDC = 총 1,000,000 mock USDC 필요. 마스터 지갑 최소 1,000개 준비 또는, Testnet 내 Transfer API로 분배 가능 여부를 사전에 확인 필요.
